# 🚀 「深教考通」高保真执行引擎 MVP 产品需求文档

## 一、 产品定位与核心哲学

* **产品形态：** 跨端应用（优先微信小程序，适配移动端碎片化交互）。
* **核心定位：** 摒弃内容分发（无内置题库/网课），专注于公考/编考人群的“备考进度追踪与高保真执行工具”。
* **核心价值观：** 拒绝情绪化焦虑，用严密的数学逻辑和结构化数据驱动复习；尊重知识本身的拓扑结构，引导用户步步为营。

---

## 二、 全局架构：漏斗式冷启动与工作区

### 1. 多目标分流与环境隔离

* **前置画像收集：** 用户首次进入需明确备考目标（如：`教师编 -> 广东省 -> 深圳市 -> 小学数学`），支持多选。
* **全局工作区 (Workspace)：** 顶部提供极简的下拉切换器。选择特定目标后，下方所有的进度流、技能树、打卡日志均实现绝对的数据隔离。
* **备考周期设定：** 用户需为当前工作区设定总复习天数（如 90 天）或具体考试日期。

### 2. 技能树引擎 (Knowledge Tree)

* **一键克隆：** 系统根据用户标签，推荐官方预埋的“同类群体通用大纲”，支持一键导入为个人实例。
* **真实层级拓扑：** 必须严格还原教材结构，支持最多四级嵌套：`L1 目标层 -> L2 模块/教材层 -> L3 章节层 -> L4 知识树叶子节点`。
* **轻量化定制：** 移动端仅允许在 L3（章节）层级下，对 L4（知识点）进行快速的文字新增或左滑删除，禁止复杂的跨层级拖拽重组。

---

## 三、 核心引擎：模块化时间箱与双轨调度算法

本模块为系统“大脑”，通过算法严格把控复习节奏。

### 1. 模块化时间箱 (Timeboxing) 与串/并行设置

* **时间下放：** 宏观的总备考天数，必须由用户下发分配至 L2/L3 的大模块中（如《教综》分配 20 天，《数学》分配 70 天）。
* **串/并行规则约束：**
* **宏观解耦（大模块级）：** 允许并行（如教综与数学同步推进）或串行（学完一本再开一本）。
* **微观锁死（DAG 严格指针）：** 在具体的 L3 章节内部，**绝对禁止打乱知识结构并行**。系统维护一个“单向推进指针”，只有当前一节知识点被标记为 `[已闭环]` 或 `[攻坚中]`，才允许解锁并派发下一节任务。



### 2. 双轨调度生成器 (Dual-Track Generator)

每日凌晨静默执行，生成次日【建议复习清单】：

* **轨道一：推进指针 (Forward Track)**
* 沿未解锁的技能树向下推进，根据模块设定的时间 Quota，稳定释放新知识点。


* **轨道二：复测回路 (Review Track)**
* 基于艾宾浩斯遗忘曲线，在后台遍历历史打卡日志。对触发遗忘阈值的老节点赋予 `P0` 最高优先级，强行塞入次日任务流中防战损。



### 3. 动态配额与航向修正 (Course Correction)

* **配额公式：** `当前活跃模块每日 Quota = 该模块剩余未闭环节点数 / 该模块分配的剩余可用天数`。
* **熔断预警（Amber Alert）：** 当计算出的 Quota 导致预估单日学习耗时严重超越用户“历史最高单日专注极值”时，在首页顶部悬浮琥珀色警告横幅。
* **策略建议：** 点击横幅提供两条降级路径：
1. 【硬扛】确认调高单日打卡时长目标。
2. 【修剪】一键将系统筛选出的“低频/边缘叶子节点”批量移入回收站，缩小分母，恢复健康配额。

## 四、 前端交互：极简执行与解耦结算（笔试版）

完全消灭“计时器”，采用全异步的动作触发与成果记录。

### 1. 首页微观压迫流 (Action Flow)

* **微型周历：** 顶部悬浮单行横向日历（周一至周日），完成当日配额即点亮绿圈，提供基础连贯性确认（Streak）。
* **今日行动卡片：** 核心视区仅陈列经系统调度派发且用户确认的 L4 知识点卡片。点击卡片即唤起半屏结算台。结算完成，卡片划线并滑出屏幕。

### 2. 动作解耦的半屏结算台 (Dynamic Bottom Sheet)

系统不预判题型，由用户选定刚刚执行的“复习动作”，动态渲染极简表单录入客观数据：

* **动作 1：客观题刷题** ➡️ 唤起数字键盘，录入 `[总题数]` 与 `[错题数]`。
* **动作 2：主观题/记忆背诵** ➡️ 唤起四挡锚点滑块：`[20% 严重漏点] / [50% 词不达意] / [80% 核心踩准] / [100% 肌肉记忆]`。
* **动作 3：材料大题/综合应用** ➡️ 唤起滑块：录入 `[有效耗时]`，及 `[核心踩分点是否全中]` (Switch 开关)。

### 3. 灵感与避坑便签 (Trap Memo)

* 结算表单最下方固定留有一个单行文本框（非必填）。
* 用户可随手记录概念陷阱或口诀（如“注意：这题考的是皮亚杰而不是维果斯基”）。提交后作为属性挂载至该节点，进入复测回路时将自动弹出作为避坑提示。

## 五、 特殊业务模块：标准化面试规范 (MVP-V1)

针对试讲等非标行为的强制约束。

* **自建目录：** 支持用户手动建立具体的试讲课文目录。
* **强制回看：** 用户在线下录制演练视频/音频后，在小程序内打卡时，必须像质检员一样逐项勾选系统锁死的结构化表单（Checklist）。
* **判定标准：** 包括且不限于：`[完成耗时（填报分钟数）]、[是否超时（红绿判定）]、[卡壳次数（滑块）]、[板书完整度]`。表单全绿方可通过并闭环该节点。

---

## 六、 数据看板：宏观多巴胺 (Dashboard)

所有延迟满足感与能力剖析收口于底部的独立 Tab。

* **全局热力月历：** 类似于 GitHub 提交记录，色块深浅与当日累积投入有效时长（或闭环节点总数）正相关。
* **结构化燃尽图：** 分模块（L2）展示技能树点亮进度的百分比进度条。
* **面试雷达图：** 聚合面试 Checklist 中的各项指标，多维展示在时间掌控、教态、板书等维度的能力极值与短板。

---

## 七、 底部导航栏 (Tab Bar) 规范

1. **🎯 执行 (Action)：** 首页。仅包含周历、行动卡片流、结算半屏弹窗。
2. **📊 看板 (Dashboard)：** 热力图与全局能力/进度透视。
3. **⚙️ 规划 (Settings)：** 工作区切换、备考周期与模块时间箱规划、技能树修剪、考纲模板管理。
