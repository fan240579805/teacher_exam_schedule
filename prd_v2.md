# 🚀 「深教考通」MVP 产品需求文档 (PRD)

## 一、 产品定位与核心理念

* **产品形态：** 跨端应用（优先微信小程序，基于 uniapp + Supabase/Hono 架构）。
* **核心定位：** 面向公考/编考人群的“备考进度追踪与高保真执行工具”。
* **核心边界：** 绝对不自带题库、网课等内容。纯粹作为用户的“执行记账本”与“外部大脑”，把复习变成像敲代码一样精确的任务闭环。

---

## 二、 核心模块 1：漏斗式冷启动与全局规划

### 1. 目标设定与工作区隔离

* **多目标支持：** 用户首次进入需选择目标（如：`广东省 -> 深圳市 -> 小学数学教师编`）。支持添加多个备考目标。
* **工作区切换 (Workspace)：** 顶部导航栏提供下拉切换。不同目标的数据（技能树、打卡日志、今日任务）完全隔离，避免上下文污染。
* **复习周期设定：** 用户需为每个目标设定考试日期或复习周期（如：90天）。

### 2. 技能树引擎 (Knowledge Tree)

* **一键克隆：** 根据用户选择的目标，系统推荐对应的“公共大纲模板”，用户一键导入作为自己的初始技能树。
* **层级结构：** 还原真实的复习材料嵌套深度（如：`科目 -> 教材 -> 章节 -> 具体知识点`）。
* **轻量化编辑：** 移动端仅支持在章节下快速新增/左滑删除具体知识点，不提供复杂的层级拖拽重组。

---

## 三、 核心模块 2：参谋调度与预警系统 (引擎层)

### 1. 动态生成建议清单

* 系统每日凌晨基于算法生成明天的“建议复习清单”。优先级排序：
* `P0`：错题重刷 / 掌握度低的防遗忘节点。
* `P1`：按艾宾浩斯曲线到期需复测的节点。
* `P2`：为满足周期进度，按序释放的全新节点。


* **用户决策：** 系统不强制派发，用户需通过“勾选”动作，将建议任务移入「明日行动卡片池」。

### 2. 动态配额与航向修正预警 (Course Correction)

* **底层公式：** `每日指标 (Quota) = 剩余未闭环节点 / 剩余备考天数`。
* **红色预警横幅：** 当系统察觉用户连续摸鱼，导致计算出的单日所需耗时远超其“历史最高单日专注时长”时，在首页顶部悬浮预警。
* **修正建议：** 用户点击预警后，提供两个挽救方案：
1. 确认强制提高每日打卡时长目标。
2. 一键将系统筛选的“低频/边缘考点”移入【战略放弃】回收站，缩小分母，恢复健康进度。

## 四、 核心模块 3：极简执行与异步结算 (笔试部分)

### 1. 首页微观压迫流 (UI 呈现)

* **微型周历：** 首页最顶部保留极简的横向周历（周一至周日），当日完成打卡即亮绿圈，提供基础连贯性情绪价值。
* **今日行动卡片：** 页面核心区域仅展示今日待办节点，无多余层级。
* **无内置计时器：** 用户在外部（纸质书/网课）完成学习后，点击卡片上的【录入成果】，弹出底部半屏结算台 (Action Sheet)。

### 2. 动作解耦的半屏结算台

知识点本身不分主客观，而是让用户在结算时选择刚刚执行了什么“动作”，动态渲染表单：

* **动作 A：客观题刷题** 👉 弹出数字键盘：填入 `[总题数]` 与 `[错题数]`（系统自动算正确率）。
* **动作 B：主观题背诵** 👉 弹出四挡明确锚点的掌握度滑块：`[20% 严重漏点] / [50% 词不达意] / [80% 核心踩准] / [100% 肌肉记忆]`。
* **动作 C：网课/泛读** 👉 弹出滑块：填入 `[耗时分钟数]`。

### 3. 灵感与避坑便签 (Memo)

* 结算表单底部固定提供一个**非强制的单行输入框**。
* 用户若有感悟，可快速记录错因或口诀（如“注意皮亚杰概念陷阱”）。该便签会挂载至该节点，下次复习时作为提示释出；若太累不想写，直接留空提交。

## 五、 核心模块 4：标准化行为规范 (面试部分)

面试模块独立于笔试逻辑，定位于非标行为的规范化。MVP 阶段仅上线 V1 纯记录版：

* **自定义目录：** 用户自行建立试讲目录（如《小学数学三年级下册》课文清单）。
* **强制回看复盘：** 用户自行用手机原相机录像/录音演练。
* **结构化自评飞行检查单 (Checklist)：** 演练后，必须对照视频逐项核对硬性指标（如：`耗时是否合规`、`口头禅是否超过5次`、`板书是否完整`等）。全绿通过才算闭环。
* *(备忘：V2 商业化版将引入录音转文本、挂载官方评分表及 LLM 大模型生成诊断报告。)*

---

## 六、 核心模块 5：宏观多巴胺 (数据看板 Tab)

将成就感数据收纳在底部的第二个 Tab，避免在首页喧宾夺主：

* **全局 GitHub 式热力图：** 以月历形式呈现，根据每日投入时长或攻克节点数，展示不同深浅的绿色色块。
* **燃尽进度条：** 展示当前科目技能树的总点亮百分比。
* **面试雷达图：** 根据 V1 版的 Checklist 记录，生成个人面试能力偏科雷达图（仪态、时间掌控、逻辑清晰度等）。

---

## 七、 底部导航栏 (Tab Bar) 结构规划

1. **🎯 执行 (Action)：** 默认首页。周历 + 今日行动卡片流 + 半屏结算台。
2. **📊 看板 (Dashboard)：** 热力月历 + 技能树全局进度透视 + 雷达图。
3. **⚙️ 规划 (Settings)：** 工作区切换、添加/修剪节点、设置复习周期、导入考纲模板。
